# baraholka-approve-bot

Бот для Telegram, выполняющий проверку пользователей (анти-бот защита) перед их вступлением в канал.

## Как это работает

### Основной сценарий

1. Пользователь отправляет запрос на вступление в канал **Барахолка**
2. Бот получает уведомление о запросе и отправляет пользователю в личные сообщения приветствие с **капчей**
3. Капча — выбор правильного эмодзи из предложенных вариантов (фрукт, животное, транспорт и т.д.)
4. При правильном ответе бот **одобряет** запрос на вступление
5. При неправильном ответе даётся ещё одна попытка (всего 3 попытки, каждая с новой капчей)
6. Если пользователь не прошёл капчу или не ответил за 5 минут — запрос **отклоняется**

### Если бот не может написать пользователю

Некоторые пользователи в настройках приватности Telegram запрещают получать сообщения от людей не из контактов. В этом случае бот не может отправить капчу автоматически. Для таких пользователей работает альтернативный сценарий:

1. Пользователь отправляет запрос на вступление → бот получает `Forbidden`
2. Бот добавляет пользователя в список ожидающих и запускает таймаут
3. Пользователь видит в описании канала ссылку на бота и переходит по ней (или пишет `/start` вручную)
4. Бот отправляет капчу, дальнейший процесс — как обычно

Для удобства пользователей в описание канала рекомендуется добавить ссылку вида:

```
Для вступления в канал пройдите проверку: https://t.me/your_bot_username?start=join
```

При клике на такую ссылку Telegram автоматически откроет бота и отправит `/start join`.

### Подсказки пользователю

Бот автоматически устанавливает описание своего профиля при запуске. Когда пользователь открывает профиль бота, он видит инструкции:

- **Краткое описание** — видно в списке чатов: «Бот-проверка для канала Барахолка. Напишите /start чтобы пройти капчу и вступить в канал.»
- **Полное описание** — видно по кнопке «Что умеет этот бот?»: подробная инструкция со ссылкой для быстрого доступа

## Предварительные требования

- Python 3.12+
- Зарегистрированный Telegram бот (через [@BotFather](https://t.me/BotFather))
- Канал Telegram с включённым режимом **«Подтверждать новых участников»**
- Доступ к `api.telegram.org` (если запускаете из РФ — может потребоваться VPN/прокси)

## Настройка бота и канала

### Шаг 1. Создание бота

1. Напишите [@BotFather](https://t.me/BotFather) команду `/newbot`
2. Укажите имя и username для бота
3. Скопируйте полученный **токен**

### Шаг 2. Настройка бота в BotFather

В BotFather → выберите бота → Bot Settings:
- **Group Privacy** → **Turn off** — чтобы бот мог видеть сообщения в группах

### Шаг 3. Настройка канала

1. Добавьте бота в канал как **администратора** с правом **«Приглашать пользователей через ссылку»**
2. В настройках канала включите **«Подтверждать новых участников»** (Join via Request)
3. Добавьте в **описание канала** ссылку для прохождения проверки:
   ```
   Для вступления в канал пройдите проверку: https://t.me/your_bot_username?start=join
   ```

## Установка и запуск

### Локально

```bash
# Клонируйте репозиторий
git clone https://github.com/your-username/baraholka-approve-bot.git
cd baraholka-approve-bot

# Создайте виртуальное окружение
python -m venv venv
source venv/bin/activate  # Linux/Mac
# venv\Scripts\activate   # Windows

# Установите зависимости
pip install -r requirements.txt

# Скопируйте и заполните .env файл
cp .env.example .env
# Отредактируйте .env, указав BOT_TOKEN и CHANNEL_ID

# Запустите бота
python main.py
```

### Через Docker

```bash
# Скопируйте и заполните .env файл
cp .env.example .env
# Отредактируйте .env, указав BOT_TOKEN и CHANNEL_ID

# Соберите и запустите
docker compose up -d

# Просмотр логов
docker compose logs -f

# Пересборка после обновления кода
docker compose up --build -d

# Остановка
docker compose down
```

> ⚠️ **Если запускаете из РФ**: Docker-контейнер должен иметь доступ к `api.telegram.org`. Если бот падает с ошибкой `Unclosed client session` — включите VPN на хост-машине или настройте прокси в Docker.

## Переменные окружения

| Переменная | Обязательно | По умолчанию | Описание |
|---|---|---|---|
| `BOT_TOKEN` | ✅ | — | Токен бота от @BotFather |
| `CHANNEL_ID` | ✅ | — | ID канала или @username |
| `WELCOME_MESSAGE` | ❌ | См. config.py | Приветственное сообщение. Поддерживает `{name}` для имени пользователя |
| `CAPTCHA_TIMEOUT` | ❌ | `300` | Таймаут капчи в секундах (5 мин) |
| `MAX_ATTEMPTS` | ❌ | `3` | Максимум попыток прохождения капчи |

## Тестирование

Для проверки работы капчи без реального запроса на вступление отправьте боту команду `/test`. Бот пришлёт капчу в тестовом режиме:

- ✅ Правильный ответ → сообщение «ТЕСТ: Капча пройдена!» (без реального одобрения)
- ❌ Неправильный ответ → новая капча, попытка списывается
- ❌ 3 неверных ответа → сообщение «ТЕСТ: Капча не пройдена» (без реального отклонения)

## Деплой на VPS

1. Подключитесь к серверу по SSH
2. Установите Docker: `curl -fsSL https://get.docker.com | sh`
3. Склонируйте репозиторий на сервер
4. Создайте `.env` файл с вашими настройками
5. Запустите: `docker compose up -d`

Бот будет автоматически перезапускаться при падениях благодаря `restart: unless-stopped`.
